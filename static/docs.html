0. 通用规则与认证

所有接口请求均需遵循以下规范。
认证方式 Header 携带：Authorization: Bearer <JWT_TOKEN>
响应格式 
{
  "code": 0,       // 0: 成功, 其他: 错误码
  "msg": "success",
  "data": { ... }  // 业务数据
}
系统角色 superadmin: 无视权限检查 | admin: 管理端接口 | staff: 员工端接口

1. 账户管理 (Account)

POST /api/login 公开
用户身份验证
Request Body: 
{
  "username": "admin",
  "password": "password123"
}
Response Data: 
{ "token": "eyJhbGciOi..." }
GET /api/admin/accounts admin
账户列表分页查询（支持 username/role 筛选）

2. 员工档案管理 (Person)

GET /api/admin/persons admin
员工分页列表。Query 参数: page, pageSize, name, dept_id
POST /api/admin/persons admin
创建新员工，同时可关联创建账户。
Request Body: 
{
  "name": "张三",
  "gender": "男",
  "birthday": "1995-01-01",
  "entry_date": "2023-10-01",
  "dept_id": 1,
  "job_title": "开发工程师",
  "create_account": true,
  "username": "zhangsan"
}

3. 部门管理 (Department)

GET /api/admin/departments admin
获取所有部门列表。
POST /api/admin/departments admin
Request Body: 
{ "name": "研发部", "manager": "李四" }

4. 人事变动 (Personnel)

POST /api/admin/personnel-changes admin
提交人事变动申请。类型 1:调岗, 2:离职, 3:部门调整。
Request Body: 
{
  "emp_id": "EMP231001",
  "change_type": 3,
  "new_value": "12", // 新部门ID或职位名称
  "reason": "业务调整",
  "execute_date": "2023-12-01"
}

5. 公告管理 (Notice)

GET /api/user/notices staff
员工查看最新公告。
POST /api/admin/notices admin
Request Body: 
{ "title": "国庆放假通知", "content": "1-7号放假...", "type": "通知" }

6. 仪表盘数据 (Dashboard)

GET /api/admin/dashboard/stats admin
返回统计数字（总人数、今日打卡率、待办审批等）。

7. 考勤管理 (Attendance)

POST /api/user/attendance/checkin staff
员工打卡（由后端判断签到/签退时间窗）。
GET /api/admin/attendance admin
全局考勤查询。参数: emp_id, date_start, date_end
PUT /api/admin/attendance/:id admin
管理员修正考勤。
Request Body: 
{
  "status": 1, // 1:正常, 2:异常, 3:迟到, 4:早退
  "check_in": "2023-10-01T09:00:00Z",
  "remark": "补签"
}

8. 权限展示 (Permission)

GET /api/user/permissions staff
获取当前用户对所有接口的权限明细，用于前端动态菜单渲染。

9. AI 聊天 (AIChat)

POST /api/chat staff
Request Body: 
{
  "message": "示例聊天内容",
  "session_id": "user-unique-session-123"
}
Response Data: 
{ "message": "示例AI返回内容" }
